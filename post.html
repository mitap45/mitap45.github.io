Php 7 ile gelen, geriye dönük uyumsuz değişiklikler

Hatalar ve istisna yönetiminde değişiklikler

muratcan itap Eylül 04, 2017

Eski php versiyonlarında bulunan ölümcül hataların çoğu Php7'de istisnalara dönüştürüldü. Bu istisnalar Php'nin çekirdek Hata sınıfından türemektedir ve bu sınıf throwable arayüzünü implemente etmektedir.

Yukarıdaki sebepten dolayı, kendi yazdığımız ve hataları kontrol edip yönettiğimiz kodlar çalışmayabilir çünkü hatalar yerine istisnalar atılıyor olabiir.

Php7'de hatalarla ilgili resmi dökümantasyona buradan ulaşabilirsiniz.

set_exception_handler() ile kayıt edilen fonksiyonlar

Eğer hatayı yakaladığımız fonksiyonu set_exception_handler() ile kayıt ediyorsak, Exception tip tanımlamasını kullanmamalıyız.

Fonksiyonun php5 ve php7'de çalışmasını istiyorsak Exception tip tanımlamasını fonksiyondan kaldırmalıyız. Eğer fonksiyon sadece php7'de çalışacaksa Exception tip tanımlaması yerine Throwable tanımlamasını kullanmalıyız. 

                // Çalışmayacak olan php5 kodu
                function handler(Exception $e) { ... }
                set_exception_handler('handler');

                // PHP 5 ve 7 için çalışır kod.
                function handler($e) { ... }

                // Sadece php7 için çalışacak olan kod.
                function handler(Throwable $e) { ... }
